Robotic Car

The project makes use of the microcontroller MPS432 to create a robotic car that is able to send request and receive data from the web portal. This data will be in the form of commands from the webportal as well as the sensors from the MSP432 that needs to be returned to the webportal.

Web Portal

This project makes use of flask, jinjja, html, javascript and CSS to build. Jinjja allows for persistent data html allows for the rendering of HTML document elements to be viewed on the website. Javascript allows logic processing of the game and provides the design of the website layout.

In order for the game to work, we researched on the different uses of the web portal and generated GET & POST request using the python libraries.

Game

This gameplay requires the student to follow the virtual maze layout and build a physical maze with the tiles provided. Once the arena is set, the student can use the web-based terminal and input the sequence of command for the car to transverse the maze.

To make the game and learning more enjoyable, the maze design to have several different routes to the end point. Upon successful completion of the maze, random bomb will be generated on the virtual map. The student will put paper cup at the tiles following the bomb location of the virtual map. The student will need to think and recalibrate the sequence of command for the car to transverse the maze while avoiding the paper cup.
